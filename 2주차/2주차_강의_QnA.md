## 👨‍🏫 강의
### 1) Primitive Type과 Wrapper Class의 차이
```
🙋‍♂️ 나:
참조형은 기본값이 Null이고, 기본형은 0인 것으로 알고 있습니다.

🧑‍🏫 멘토:
int는 객체일까요?

🙋‍♂️ 나:
객체는 아닙니다.

🧑‍🏫 멘토:
맞아요 객체가 아니라 null같은게 바인딩이 안되고, 실제로 스택에 들어가게 되어 메서드가 종료될 때 소멸하게 되고,
래퍼클래스는 힙에 저장이 되어서 소멸은 GC가 해준다.
래퍼와 프리미티브의 덧셈이 되는가?

🙋‍♂️ 나: 됩니다. 래퍼클래스로 변경하는걸 박싱, 프리미티브로 변경하는걸 언박싱이라고 부릅니다.
```

### 2) SET
```
🧑‍🏫 멘토:
SET은 컬렉션인데 그 컬렉션들끼리 같은 객체인지 비교를 어떻게 할까요?

🙋‍♂️ 나:
잘 모르겠습니다. equals를 사용하나요?

🧑‍🏫 멘토:
contains()라는 메서드가 있는데 이걸로 어떻게 비교를 할까요?

🙋‍♂️ 나:
hash?를 사용했던 것 같아요

🧑‍🏫 멘토:
equals, hashCode에 대해서 아실까요? 얘네는 동등성을 비교하는 것이다.
equals는 두 객체가 같은지 보는거고
hashCode는 set같은 컬렉션에서 얘네가 같은지 비교하는 것이다.
해시코드는 메모리 값을 기본적으로 갖고있어서 중요한데 이 부분은 깊게 공부를 좀 하셔야합니다.
그래서 equals와 hashCode를 함께 재정이 해야합니다.
```
